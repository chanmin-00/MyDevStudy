# 스코프

스코프란 변수, 함수, 클래스와 같은 멤버들을 서로 공유하여 사용할 수 있는 범위를 지정하는 단위이다.

스코프가 지정되는 범위에는 크게 패키지 내부, 클래스 내부, 함수 내부 등이 있다.

### **스코프 규칙 3가지**

1. **스코프 외부에서는 내부 멤버를 참조 연산자로만 접근 가능**
2. **동일 스코프 내에서는 멤버들을 공유 가능**
    
    ```kotlin
    val a = "패키지 스코프"
    
    class B {
    		fun print() {
    				println(a)
    		}
    }
    
    fun main() {
    		println(a)
    		B().print()
    }
    ```
    
    - 패키지를 명시하지 않았으므로 모두 default 패키지에 속한다.
    - 동일한 스코프 내에서는 변수를 공유할 수 있다.
    
    하위 스코프에서는 상위 스코프의 멤버를 재정의할 수 있다. 스코프의 같은 레벨에서는 같은 이름의 멤버를 만들어서는 안되는데, 하위 스코프에서는 같은 이름의 멤버를 만들어 사용할 수 있다. 
    
    ```kotlin
    var a = "~~~"
    var a = "dkfjdkf"
    
    // 선언부가 서로 충돌하는 에러가 발생하게 됨
    ```
    
    ```kotlin
    val a = "패키지 스코프"
    
    class B {
    		val a = "클래스 스코프"
    		
    		fun print() {
    				println(a)
    		}
    }
    
    fun main() {
    		val a = "함수 스코프"
    		println(a)
    		B().print()
    }
    ```
    
    - 상위 스코프에 있던 변수를 참조하지 않고  각각의 스코프에 있는 값을 참조함

# 접근 제한자

스코프 외부에서 내부 멤버에 접근할 수 있는 권한을 제어하는 기능이다. 코틀린에서는 public, internal, private, protected 네 가지 접근 제한자를 제공한다.

### **1. 패키지 스코프에서의 접근 제한자**

- public (기본값): 어디서든 접근 가능
- internal : 같은 **모듈 내**에서만 접근 가능
- private : 같은 **파일 내**에서만 접근 가능
- protected : 사용하지 않음

### **2. 클래스 스코프에서의 접근 제한자**

- public (기본값): 클래스 외부에서 항상 접근 가능
- private : 클래스 내부에서만 접근 가능
- protected : 클래스 내부 + 상속받은 클래스 내부에서 접근 가능
- internal : 사용하지 않음
