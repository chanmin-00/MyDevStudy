# 고차함수

고차함수란 함수를 값처럼 다루는 방법이다. 즉, **함수를 다른 함수의 파라미터로 전달하거나, 함수의 결과값으로 반환할 수 있는 함수**를 말한다.

코틀린에서는 **모든 함수가 고차함수로 사용 가능**하다.

```kotlin
fun main() {
    b(::a)
}

fun a(str: String) {
    println("$str 함수 a")
}

fun b(function: (String) -> Unit) {
    function("b가 호출한")
}
```

- b는 함수 a를 함수형 파라미터로 전달받는다.
- `(String) -> Unit` 은 “문자열을 받아 반환값이 없는 함수”라는 의미이다.
- `Unit` 은 값이 없음을 의미하며, Java의 void와 유사하다.
- `::a` 는 함수 참조 연산자로, 함수를 그대로 넘길 때 사용한다.

```kotlin
b가 호출한 함수 a
```

## 함수 자료형

함수의 형식을 파라미터 자료형처럼 명시할 수 있다.

```kotlin
fun b(function: (String) -> Unit) { ... }
```

이 의미는 "문자열을 입력받고, 반환값이 없는 함수"를 파라미터로 받겠다는 것이다. 이렇게 작성하면 동일한 형식의 함수를 모두 전달할 수 있다.

# 람다 함수

함수를 굳이 이름 붙여 따로 만들 필요가 없다면, **람다식**으로 표현할 수 있다. 람다함수는 그 자체로 고차함수이며, 변수에 직접 담을 수도 있다.

```kotlin
fun main() {
    b(::a)

    val c: (String) -> Unit = { str -> println("$str 람다함수") }
    b(c)
}

fun a(str: String) {
    println("$str 함수 a")
}

fun b(function: (String) -> Unit) {
    function("b가 호출한")
}
```

```kotlin
b가 호출한 함수 a
b가 호출한 람다함수
```

## **람다 표현식 단축**

람다식에서는 **타입추론**이 가능하기 때문에, 더 간결하게 작성할 수 있다.

```kotlin
val c = { str: String -> println("$str 람다함수") }
```

- `(String) -> Unit` 형식으로 자동 추론된다.
- `{ str -> ... }` 형태로 파라미터 타입을 생략할 수도 있다.
